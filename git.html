Що таке система контролю версій?

Система контролю версій (СКВ) — це програмний інструмент, що допомагає відстежувати та зберігати історію змін у файлах, координувати роботу над проєктом у команді та уникати втрати даних.

Історія розвитку СКВ

  • Покоління 1: локальні системи — зміни зберігались у вигляді копій файлів (наприклад, RCS).
  • Покоління 2: централізовані системи — один сервер із “центральним” репозиторієм (CVS, SVN).
  • Покоління 3: розподілені системи — кожен має повну копію проєкту (Git, Mercurial).

Приклади систем

  • CVS — старіша централізована система.
  • SVN — корпоративна централізована СКВ.
  • Mercurial — альтернатива Git.
  • Perforce — комерційна СКВ.
  • Git — сучасний стандарт.

Чому саме Git?

Git створив Лінус Торвальдс у 2005 році для ядра Linux. Сьогодні Git — де-факто стандарт і основа для GitHub, GitLab, Bitbucket.

Переваги використання Git

  • Збереження історії змін.
  • Паралельна робота розробників.
  • Швидке повернення до попередніх версій.
  • Робота офлайн.
  • Безкоштовний і відкритий код.

10 базових команд Git

git init — створення нового репозиторію.
git clone <url> — клонування існуючого репозиторію.
git status — перевірка стану файлів.
git add <file> — додавання файлу до індексу.
git commit -m "повідомлення" — збереження змін із коментарем.
git log — перегляд історії комітів.
git branch — робота з гілками.
git checkout <branch> — перехід на іншу гілку.
git merge <branch> — об’єднання гілок.
git push / git pull — відправка/отримання змін.
