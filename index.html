Основи роботи з Git

Що таке система контролю версій?

Система контролю версій (СКВ) — це програмний інструмент, що допомагає відстежувати та зберігати історію змін у файлах, координувати роботу над проєктом у команді та уникати втрати даних. Вона дозволяє “повертатися назад у часі” — переглядати старі версії коду, виправляти помилки і працювати паралельно над різними частинами проєкту.

Історія розвитку СКВ

  • Покоління 1: локальні системи — зміни зберігались у вигляді копій файлів (наприклад, RCS). Це підходило лише для індивідуальної роботи.
  • Покоління 2: централізовані системи — з’явився один сервер із “центральним” репозиторієм (CVS, SVN). Мінус — якщо сервер падав, робота зупинялась.
  • Покоління 3: розподілені системи — кожен розробник має повну копію історії проєкту (Git, Mercurial). Це забезпечує надійність і швидкість.

Приклади систем контролю версій

  • CVS (Concurrent Versions System) — одна з найстаріших СКВ, використовувалась у 90-х.
  • SVN (Subversion) — централізована СКВ, поширена у корпоративному середовищі.
  • Mercurial — альтернатива Git, проста у використанні, застосовувалась у Mozilla.
  • Perforce — комерційна СКВ, популярна у великих ігрових студіях.
  • Git — сучасна розподілена система, яка стала стандартом у світі розробки.

Чому саме Git?

Git був створений у 2005 році Лінусом Торвальдсом для роботи над ядром Linux. Він поєднав у собі швидкість, надійність, гнучкість та простоту в роботі з гілками (branches). На відміну від SVN чи CVS, Git працює навіть без постійного підключення до мережі, а його архітектура дозволяє тисячам розробників ефективно співпрацювати над одним проєктом.

Сьогодні Git став де-факто стандартом для контролю версій, і саме на ньому базуються популярні платформи GitHub, GitLab, Bitbucket.

Переваги використання Git

  • Збереження історії змін.
  • Паралельна робота декількох розробників.
  • Можливість швидкого повернення до попередніх версій.
  • Робота офлайн (локальне збереження).
  • Безкоштовність і відкритий код.

10 базових команд Git

git init — створення нового репозиторію.
git clone <url> — клонування існуючого репозиторію.
git status — перевірка стану файлів.
git add <file> — додавання файлу до індексу.
git commit -m "повідомлення" — збереження змін із коментарем.
git log — перегляд історії комітів.
git branch — робота з гілками.
git checkout <branch> — перехід на іншу гілку.
git merge <branch> — об’єднання гілок.
git push / git pull — відправка/отримання змін із віддаленого репозиторію.
